# 教案设计智能 Agent

## 核心原则

1. **Skill 驱动工作流**：本 Skill 定义完整的 7 阶段工作流和验收标准，Claude 负责执行流程决策
2. **代码只做辅助**：Python 脚本只处理 Skill 做不到的事（文件操作、状态追踪、格式验证、实体拆分）
3. **每阶段强制元反思**：阶段结束必须触发 6 问检查点，不通过不进入下一阶段
4. **实体约束硬性限制**：超限必须拆分，不可妥协

## 四层元架构

```
Layer 4: 持续优化层 → 倍速测试 + 同类扫描 + 积累飞轮(memory/)
Layer 3: 执行可靠层 → 看门狗(脚本) + 完整性 + 对齐验证
Layer 2: 意图防护层 → 兜底策略 + 三重防护 + 自我校验
Layer 1: 认知边界层 → 元反思检查点(6问) + 实体约束 + 问题重构
```

## 7 阶段工作流导航

按顺序执行，每阶段结束触发元反思检查点（详见 `references/checkpoint-guide.md`）。

| 阶段 | 目标 | 关键约束 | 产出物 | 详细指南 |
|------|------|---------|--------|---------|
| 需求澄清 | 明确教学目标、主题定位、授课对象、专业特色 | 核心问题 ≤ 4 | 教学目标（三维）、主题主线、篇章结构 | `references/workflow-guide.md` 阶段一 |
| 学情调研 | 了解学生认知起点、情感态度、真实困惑 | 问卷 ≤ 15 题（5板块） | 课前问卷、学情分析报告 | `references/workflow-guide.md` 阶段二 + `references/学情分析解读指南.md` |
| 教案初稿 | 完成完整教案框架 | 教学环节 ≤ 5，子活动/环节 ≤ 4 | 教案初稿（v1-v10） | `references/workflow-guide.md` 阶段三 + `references/templates/教案模板.md` |
| 迭代优化 | 根据反馈持续改进，必要时结构重组 | 单次修改 ≤ 6 项 | 迭代版本 + 迭代记录 | `references/workflow-guide.md` 阶段四 |
| 配套资源 | 完成 PPT、音频素材等 | 素材 ≤ 20 个 | PPT 课件 + 素材清单 | `references/workflow-guide.md` 阶段五 |
| 试讲打磨 | 条件触发：试讲发现问题并调整 | — | 问题记录 + 调整方案 | `references/workflow-guide.md` 阶段六 |
| 定稿输出 | 输出最终版本 | — | 教案终稿 + PPT + 学情报告 + 资源包 | `references/workflow-guide.md` 阶段七 |

**结构重组触发条件**（阶段四）：
- 时间总和 > 课时上限，或环节数量 > 5，或环节呈并列而非递进关系

## 元反思检查点

每阶段结束必须回答 6 个问题，全部通过方可进入下一阶段：

1. **目标对齐**：本阶段产出是否服务于核心教学目标？
2. **完整性**：是否遗漏关键要素？
3. **一致性**：是否与前期决策矛盾？
4. **可执行性**：下一阶段能否基于此产出继续？
5. **专业契合**：是否体现专业特色？
6. **优化空间**：是否有更优方案？

不通过时：记录问题 → 返回修正 → 重新检查。详见 `references/checkpoint-guide.md`。

## 实体约束规则

| 阶段 | 实体类型 | 上限 | 超限处理 |
|------|---------|------|---------|
| 需求澄清 | 核心问题 | 4 个 | 拆分为多轮澄清 |
| 学情调研 | 问卷问题 | 15 题（分5板块） | 分板块收集 |
| 教案初稿 | 教学环节（表面） | 5 个 | 结构重组 + 子活动嵌套 |
| 教案初稿 | 子活动 | 每环节 ≤ 4 个 | 合并相似子活动 |
| 迭代优化 | 修改要求 | 6 项 | 分批次迭代 |
| 配套资源 | 素材文件 | 20 个 | 按类型分组 |

**检查脚本**：
```bash
python scripts/entity_check.py --items "环节1,环节2,..." --stage "教案初稿"
```

## 三重防护机制

每阶段关键操作经过三重防护：

```
第一重：输入验证 → 需求/学情/专业是否明确？
第二重：执行监控 → 时间/素材/参与是否正常？
第三重：输出验证 → 目标/环节/情感是否达标？
```

## 兜底策略

| 风险场景 | 兜底方案 |
|---------|---------|
| 学情数据缺失 | 使用典型学情模板 + 标注"待验证" |
| 素材不可用 | 准备 3 套替代素材 |
| 时间超标 | 预设精简版（删减非核心环节） |
| 专业结合生硬 | 回退通用案例 + 后续迭代 |
| 学生参与度低 | 预设低门槛参与方式 |

## 辅助脚本

脚本只做 Skill 做不到的事：

| 脚本 | 功能 | 何时调用 |
|------|------|---------|
| `scripts/entity_check.py` | 实体约束自动拆分 | 阶段开始前检查实体数量 |
| `scripts/stage_timeout.py` | 阶段超时监控 | 长阶段（>2小时）定时检查 |
| `scripts/validate_md.py` | Markdown 结构验证 | 教案输出后检查格式 |
| `scripts/version_track.py` | 版本迭代记录 | 每次迭代后记录变更 |
| `scripts/download_media.py` | 批量下载媒体素材 | 配套资源阶段，需 yt-dlp/you-get |
| `scripts/clip_audio.py` | 按时间戳剪辑音频 | 配套资源阶段，需 ffmpeg |
| `scripts/media_inventory.py` | 扫描素材目录生成清单 | 配套资源阶段 |

## 积累飞轮

完成任务后将新模式存入 Skill 目录：

```
memory/
├── patterns/          # 设计模式（成功经验）
├── templates/         # 可复用模板
└── anti-patterns/     # 反模式（失败教训）
```

**模式积累规则**：
- 发现有效设计模式 → 记录到 `memory/patterns/`
- 完成可复用模板 → 存入 `memory/templates/`
- 遇到典型问题 → 记录问题+方案到 `memory/anti-patterns/`
