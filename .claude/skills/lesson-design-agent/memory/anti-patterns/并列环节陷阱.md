# 并列环节陷阱

> 来源：常见问题清单问题5（`references/anti-patterns/常见问题清单.md`）
> 典型案例：v25（10 环节并列）→ v27（3+1 环节递进）

---

## 症状识别

- 每个环节独立成章，缺乏递进关系
- 学生感受："讲了很多东西，但不知道重点"
- 时间总和远超课时上限
- 用"历史线"而非"学习线"组织内容

## 根因诊断

- 每个环节都在"告诉学生知识"，而非"引导学生体验"
- 缺乏结构诊断和主线提炼
- 信息贪多，每个环节都舍不得删

## 修复方法

| 方法 | 操作 | 效果 |
|------|------|------|
| 主线提炼 | "篇章并列"→"主线递进" | 逻辑清晰，重点突出 |
| 环节合并 | 服务于同一目标的环节合并 | 减少表面环节数 |
| 子活动嵌套 | 大环节内设 2-4 子活动 | 内容丰富但不臃肿 |

## 典型案例

```
原：导入→建党→连线→改革→强国→榜样→讨论→朗诵→总结→作业（10环节并列）
改：导入→历史的声音（内含4子站）→榜样的声音（内含4关卡）→青年的声音（内含承诺+朗诵）
结果：表面环节从10个压缩到4个，主线从"历史罗列"变为"声音递进"
```
