# 结构诊断工作流

> 诊断教案的结构问题并提供重组建议。可从主流程调用，也可独立运行。

## 何时运行 / 何时跳过

**运行条件**（满足任一）：
- 用户已有教案，需要诊断结构问题
- 时间总和 > 课时上限
- 环节数量 > 5
- 环节之间呈并列关系而非递进关系

**跳过条件**：
- 教案结构已确认合理

## 前置条件

- [ ] 已有教案文件（至少 v1 版本）

## 步骤

### 1. 读取教案

读取现有教案文件，提取结构信息。

### 2. 结构验证

运行 `scripts/validate_md.py` 验证基本结构：

```bash
python scripts/validate_md.py --file "教案.md"
```

检查项：
- 必需章节是否齐全（7 个）
- 每环节是否有三要素
- 环节数量是否 ≤ 5
- 子活动数量是否 ≤ 4

### 3. 递进关系检查

绘制环节关系图，判断：
- 环节之间是**并列**还是**递进**？
- 能否用一句话概括整节课的推进逻辑？

**并列特征**：各环节独立，可调换顺序，服务同一目标
**递进特征**：后一环节建立在前一环节基础上，有明确的推进逻辑

### 4. 实体约束检查

运行 `scripts/entity_check.py` 验证实体数量：

```bash
python scripts/entity_check.py --items "环节1,环节2,..." --stage "教案初稿"
```

### 5. 输出诊断报告

输出结构化诊断报告，包含：
- 结构验证结果
- 递进/并列判断
- 实体约束检查结果
- 重组建议（如需要）

重组建议参考 `references/workflow-guide.md` 阶段四的 4.1-4.3 节。

## 独立运行说明

本工作流可脱离主流程独立运行。前提：提供已完成的教案文件路径。
