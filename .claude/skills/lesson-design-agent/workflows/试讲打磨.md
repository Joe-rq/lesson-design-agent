# 试讲打磨工作流

> 通过试讲发现问题并调整教案。可从主流程调用，也可独立运行。

## 何时运行 / 何时跳过

**运行条件**（满足任一）：
- 用户明确要求试讲
- 首次使用此教案
- 教案涉及新的教学形式（模拟人生、角色扮演等）

**跳过条件**（满足任一）：
- 用户表示"直接定稿"
- 迭代轮次 > 5（已充分打磨）
- 成熟教案的微调版本

## 前置条件

- [ ] 教案已完成（至少有 v1 版本）
- [ ] PPT 或教学材料已准备（可选）

## 步骤

### 1. 试讲执行

按教案实际授课或模拟授课，记录每个环节的实际用时。

### 2. 五维检查

逐项检查并记录：

| 维度 | 检查内容 | 状态 |
|------|---------|------|
| 时间 | 每个环节是否超时？ | [ ] |
| 素材 | 音频/视频是否正常播放？ | [ ] |
| 互动 | 学生活动是否有效参与？ | [ ] |
| 逻辑 | 环节过渡是否自然流畅？ | [ ] |
| 情感 | 课堂氛围是否到位？ | [ ] |

### 3. 问题诊断

对照 `references/anti-patterns/常见问题清单.md` 诊断，分类记录问题：
- 内容超载
- 参与不足
- 专业结合生硬
- 情感升华不到位

### 4. 调整方案

每个问题对应一个调整方案。调整方案受 design_lock.md 约束，不可违反锁定的核心决策。

### 5. 完成检查点

⛔ **BLOCKING**：展示五维检查结果和调整方案，等待用户确认后继续。

## 独立运行说明

本工作流可脱离主流程独立运行。前提：提供已完成的教案文件路径。
